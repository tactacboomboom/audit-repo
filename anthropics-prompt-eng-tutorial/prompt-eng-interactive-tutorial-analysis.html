prompt-eng-interactive-tutorial — Analyse

Anthropic officiel · ~35.8k stars · branche master

Le décor

Un repo officiel signé Anthropic, vieux de plusieurs versions de Claude mais constamment plébiscité (35 800 étoiles). Treize notebooks Jupyter dans un dossier nommé sobrement Anthropic 1P/, une variante miroir pour AmazonBedrock/, et un fichier d'indices Python hints.py. C'est un cours, pas une bibliothèque. La granularité est pédagogique : chaque leçon enseigne UN pattern, immédiatement testable dans une cellule de code.

Les acteurs sont trois : l'apprenant qui modifie des prompts, le helper get_completion() qui masque l'API, et Claude lui-même qui répond. Pas de framework, pas de wrapper, rien à apprendre côté code. Le seul objet manipulé, c'est le texte du prompt.

Les règles du jeu

Le cursus impose un ordre. On n'apprend pas la chain-of-thought avant la structure de base d'un message. On apprend d'abord que la liste messages doit alterner user/assistant et démarrer par user. Puis qu'on parle clairement. Puis qu'on assigne un rôle. Puis qu'on sépare les données des instructions avec des balises XML. Puis qu'on peut parler à la place de Claude. Puis qu'on demande à Claude de penser avant de répondre. Puis qu'on lui donne des exemples. Puis qu'on évite les hallucinations. Puis et seulement puis, on compose un prompt complexe complet.

"If a colleague is confused by your prompt, Claude's confused." — la Golden Rule, leçon 02.

La tension principale du cursus est entre contrôle et liberté. Plus on encadre Claude (rôle, format, exemples, prefill), plus la sortie devient prédictible — mais plus le prompt devient long. Anthropic ne cache pas ce tradeoff : la leçon 09 énumère dix éléments possibles tout en précisant qu'ils ne sont pas tous nécessaires.

Le socle immuable

I1Les messages alternent user/assistant et commencent par user. Toujours.
I2Les XML tags sont LA convention de séparation. Claude a été entraîné spécifiquement pour les reconnaître.
I3"Thinking only counts when it's out loud." La CoT silencieuse n'existe pas.
I4La tâche immédiate se place à la fin du prompt — biais de récence.
I5Le système est facultatif, le user obligatoire. Le system est un paramètre, pas un message.

Les leviers de transformation

Sept leviers émergent du cursus :

  • Rôle — change le style ET l'exactitude. "You are a cat." reformule. "You are a logic bot." améliore la précision.
  • Tone — séparé du rôle, pour qu'un même rôle puisse adopter plusieurs registres.
  • Few-shot — deux ou trois exemples valent mieux qu'un paragraphe d'instructions.
  • Prefill — placer { ou <tag> dans le tour assistant force le format.
  • Chain-of-thought — demander explicitement <brainstorm> ou <positive-argument> avant la réponse.
  • Give-an-out — autoriser "I don't know" supprime l'incitation à halluciner.
  • Chaining — découper en étapes, l'output N devient l'input N+1.

Les points de rupture

Le tutorial est daté de Claude 3 Haiku. Le format XML de tool use enseigné en appendice 10.2 a été remplacé en production par le tool use API natif. C'est le seul point où le contenu vieillit visiblement. Tout le reste — la Golden Rule, le framework 10-éléments, les XML tags, le prefilling — reste valide en 2026.

Le chemin de moindre résistance

clone→ pip install anthropic→ notebook 00→ API key→ 01 → 09 séquentiel→ 10.1-10.3 optionnel

Le happy path est strictement linéaire. Pas de prérequis hors-cursus, pas de configuration complexe, pas d'environnement Docker. Une clé API, Python, c'est tout.

L'équilibre du risque

Pour le skill architecture-anthropic, ce repo est une source d'autorité. Le risque d'obsolescence est faible : Anthropic continue de référencer ces patterns dans sa doc récente. Le risque de contamination par contenu non vérifié est nul : c'est le repo officiel de l'éditeur. Le bénéfice de citation est élevé : on peut écrire "ainsi Anthropic recommande de prompter dans son tutoriel officiel" sans glose.

L0Ontologie

Six primitives non-redondantes :

  • Lesson — unité pédagogique numérotée (1 fichier .ipynb)
  • Prompt — chaîne textuelle envoyée à Claude
  • Message — tuple (role, content) avec role ∈ {user, assistant}
  • Tag — délimiteur XML entourant du contenu
  • Example — paire input/output démonstrative dans le prompt
  • Variable — input externe substitué dans le template

L1Ensembles et relations

Lesson ⊂ Cursus
Prompt = Template ⊕ Variable
Message ∈ Conversation
Tag ⊂ Prompt
Example ⊂ Prompt
Conversation = [user, assistant, user, ...]

L2Logique et connecteurs

  • Prompt clair AND rôle assigné → meilleure performance
  • Données séparées XOR concaténation libre (jamais les deux)
  • CoT OR few-shot — pas toujours nécessaire de combiner
  • Prefilling AND tag d'ouverture → format déterministe
  • Anti-hallucination = (give an out) OR (evidence first) OR (temp=0)

Tension principale : contrôle (prompt long) ⊥ concision (prompt court).

L3Invariants

I1Messages alternent user/assistant, commencent par user
I2XML tags = convention native de séparation pour Claude
I3Thinking out loud only — pas de CoT implicite
I4Recency bias — tâche immédiate en fin de prompt
I5System param ≠ message (paramètre séparé de l'API)
I6Helper unique get_completion() immutable côté apprenant
